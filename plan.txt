ZYRA PROJECT PLAN

1. Project Purpose

This project is an AI-based hiring and assessment platform.
It supports:
- candidate application with resume upload
- ATS-based resume analysis
- automatic shortlist or rejection logic
- recruiter/admin dashboard
- candidate login with limited credential usage
- MCQ assessment round
- AI avatar virtual interview round
- AI-generated interview reporting
- email notifications at important stages


2. Main Stack Used

Backend:
- Python
- Flask
- PyMongo / MongoDB Atlas
- Cloudinary for resume storage
- SMTP / Gmail for email sending

Frontend:

- stores shortlisted candidates

- stores generated MCQ tests

- stores available job roles

5. Main Business Flow

Step 1: Candidate applies
- candidate uploads PDF resume
- form posts to /api/apply
- resume text is extracted
- PDF is uploaded to Cloudinary
- ATS logic analyzes the resume against selected job

Step 2: ATS decision
- shortlisted: candidate account is auto-created
- rejected: application is marked rejected
- review: application stays pending

Step 3: Email notifications
- shortlisted candidates get login credentials by email
- rejected candidates get rejection email
- after virtual interview submission, candidate gets thank-you email

Step 4: Candidate login
- candidate logs in through /api/candidate/login
- credentials can be used only 2 times
- after 2 uses, login is blocked and user must contact customer care

Step 5: MCQ round
- candidate starts MCQ test
- questions are generated dynamically
- question duplication is reduced using variation seed + recent test history
- score is calculated on submit
- if score >= 60%, candidate is auto-promoted to virtual round

Step 6: Virtual AI interview
- candidate starts AI avatar interview
- webcam/mic preview is used
- AI responds after each answer
- next question difficulty adapts based on answer quality
- final report is generated after submission

Step 7: Recruiter/Admin review
- HR dashboard shows:
  - pending candidates
  - round 1 candidates
  - round 2 candidates
  - rejected candidates
- admin can open result modal
- admin sees ATS details, MCQ details, virtual feedback, and AI interview report


6. ATS Logic

Main function:
- analyze_resume_payload(...)

What it uses:
- required skill match
- preferred skill match
- role alignment
- experience hints in resume text
- evidence signals in resume text

Output:
- score out of 100
- decision:
  - shortlisted
  - review
  - rejected
- summary
- breakdown details

Important note:
- ATS now supports multiple professional categories, not only coding roles

8. MCQ Logic

Main routes:
- /api/start_test
- /api/submit_test

Generation logic:
- generate_mcq_questions_with_fallback(...)
- generate_deterministic_mcq(...)

How it works:
- AI generation is attempted first
- prompt includes role, skills, and variation seed
- recent test questions are excluded to reduce duplication across candidates
- if AI fails, deterministic fallback generates profession-aware questions

Scoring:
- correct answers are counted
- score out of 10 is computed
- percentage is computed

Promotion rule:
- if MCQ score percent >= 60
- candidate is automatically promoted to virtual round
- no human action required


9. Virtual Interview Logic

Main routes:
- /api/virtual/questions
- /api/virtual/respond
- /api/virtual/submit

How it works:
- system generates a set of virtual questions
- candidate answers through typing or speech capture
- AI interviewer returns short feedback after each answer
- answer metrics are computed locally and/or through AI
- next question difficulty is adapted

Difficulty levels:
- foundation
- intermediate
- advanced

Adaptive logic considers:
- semantic relevance
- NLP quality
- confidence
- problem solving
- communication


10. AI Interview Report Logic

- confidence
- problem_solving
- communication
- overall_performance
- difficulty_progression
- performance_summary
- strengths
- improvements
- recommendation
- final_score

Important behavior:
- candidate should not see this report
- admin/recruiter can see this report in View Result


11. Candidate Login Logic

Route:
- /api/candidate/login

How it works:
- username and password are checked
- password is hashed in DB
- credential_login_limit defaults to 2
- credential_login_count increments on each successful login
- after limit is exceeded, login is blocked


12. Recruiter/Admin Logic

Route:
- /api/admin/login

Admin dashboard capabilities:
- create jobs
- review ATS results
- accept/reject pending applications
- promote to round 2
- reject after MCQ
- open candidate result modal

Result modal shows:
- ATS score and summary
- MCQ score and answered count
- virtual score and feedback
- AI interview report
- resume link


13. Proctoring Logic

Assessment proctoring is used in:
- MCQ interview
- virtual interview

Checks include:
- tab hidden
- window blur
- fullscreen exit
- right click blocked
- copy/cut blocked
- text selection blocked
- common copy shortcuts blocked

Violation policy:
- MCQ: auto-submit on third violation
- Virtual: auto-submit on second violation

17. Security / Behavior Notes

- passwords are hashed for candidate users
- admin login uses env-based credentials
- session-based login is used
- candidate credentials are limited to 2 successful uses
- right-click/copy restrictions are enforced during assessments


18. Deployment Recommendation

Best deployment target for current architecture:
- Google Cloud Run

Why:
- Flask backend fits better than Netlify/Vercel
- supports long backend flows better
- easy to connect Secret Manager, Artifact Registry, and external APIs

Recommended deploy setup:
- Dockerfile
- gunicorn
- Cloud Run service
- Secret Manager for all secrets
- keep MongoDB Atlas


19. Known Practical Notes

- local mock frontend logic still exists as fallback if backend request fails
- real behavior should come from Flask backend
- Ollama local mode is useful in local development but cloud deployment should prefer API-based AI providers unless separate model hosting is planned
- existing old Mongo job records may not auto-refresh seeded jobs unless collection is empty
